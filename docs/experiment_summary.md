# 主线实验总结

本文档概述最终开源主线相对官方 baseline 的形成过程。更完整的实验目录见 [experiment_catalog.md](./experiment_catalog.md)。

## 官方 baseline

官方 baseline 是本仓库的对照起点，对应公开 AUC 约 `0.812`。本开源包不是官方 baseline 的再发布，而是在相同任务口径下整理出的改进版本。

后续阶段的意义主要在于回答一个问题：相对官方 baseline，哪些信息路径和训练策略真正带来了公开榜提升。

## 阶段一：结构化处理用户侧 dense 特征

早期实验显示，相对官方 baseline 中较直接的 dense 使用方式，用户侧 dense 特征更适合先按字段结构处理，再进入主干模型。

主线方案因此转向“先按字段结构整理，再形成单个用户侧 dense 表示 token”的方式。后续实验进一步说明，直接增加 dense token 数、扩宽 hidden 维度或提高 dropout，都会明显扰动当前主干的 token 几何。

## 阶段二：补回样本级摘要信息

后续加入了样本级摘要信息，包括序列长度、截断强度、稀疏/缺失模式、recency 和 span 等。

这类信息描述的是当前样本所处状态。它不是替代序列建模，而是补充固定长度序列、padding 和 sentinel 处理过程中容易被压平的信息。

| 阶段 | 公开 AUC |
| --- | ---: |
| 样本级摘要特征工程版本 | `0.816` |

相对官方 baseline，该阶段带来约 `+0.0040` 的公开 AUC 提升。它是后续 `v24` 的重要父线：先验证 meta-summary 特征本身有效，再进一步优化这些信息进入模型的位置。

## 阶段三：让元信息提前参与 query 生成

在样本级摘要特征工程基础上，元信息从末端残差位置前移到 query 生成阶段。

| 阶段 | 公开 AUC |
| --- | ---: |
| 早期样本级元信息条件化版本 | `0.821977` |

相对官方 baseline，该阶段带来约 `+0.0100` 的公开 AUC 提升；相对样本级摘要特征工程父线也继续提升约 `+0.0060`。该结果说明，元信息更适合参与表示提取过程，而不是只作为最终打分前的校正信号。

## 阶段四：加入离散日历时间特征

下一次明确提升来自粗粒度离散日历时间：

- hour one-hot；
- weekday one-hot；
- hour-weekday one-hot。

| 阶段 | 公开 AUC |
| --- | ---: |
| 离散日历时间版本 | `0.825905` |

相对官方 baseline，该阶段提升约为 `+0.0139`。这些特征沿用样本级元信息通道，没有形成独立的强时间分支。后续更强时间建模实验没有继续带来稳定收益。

## 阶段五：强化训练策略

最终公开版本保持离散日历时间版本的模型和特征主线不变，只合入训练侧策略：

- `bce_pairwise`；
- pairwise lambda；
- cosine 学习率调度；
- warmup；
- EMA；
- label smoothing；
- dense weight decay。

| 阶段 | 公开 AUC |
| --- | ---: |
| 训练策略强化版本 | `0.828537` |

相对官方 baseline，最终公开版本提升约 `+0.0165`。这一步说明，旧主线并非主要缺少更复杂的特征或更大的模型，而是仍存在训练和正则化空间。

## 总结

相对官方 baseline，公开主线的提升来自两类改动：

1. 补回固定长度序列和缺失值处理过程中容易丢失的样本级摘要信息。
2. 将已有信息放到更合适的位置，包括结构化 dense 表示、早期元信息条件化和粗粒度时间上下文。
3. 在模型主干稳定后改进训练策略，而不是继续增加结构复杂度。

最终版本应理解为“信息接入位置 + 训练策略”的组合，而不是某一个单独技巧。
